# 客户端和服务端怎么交流

本点播机设计为可以有多个客户端与一个服务端交流，这样子不同主播就可以放同一个歌单了。

## 信息类型

客户端（也就是前端）和服务端（后端）只这交流下面这几种信息。

### `ClientBeg`

- 客户端刚上线的时候

表示客户端没在放歌，求求服务端把歌单发过来。

### `ClientEnd(song)`

- 客户端放完歌之后

说当前歌曲 `song` 已经放完，这样服务端就可以发送播放下一首歌的信息。

### `ServerSongs(list)`

- 回应 `ClientBeg` 的时候
- 服务端结束了一首歌的时候
- 服务端添加了一首歌的时候
- 服务端取消了一首歌的时候

服务端告诉客户端当前歌单是 `list` 。
