# 客户端和服务端怎么交流

本点播机设计为可以有多个客户端与一个服务端交流，这样子不同主播就可以放同一个歌单了。

## 信息类型

客户端（也就是前端）和服务端（后端）只这交流下面这几种信息。

### `ClientEnd`

- 客户端放完歌之后

说当前歌曲已经放完，这样服务端就可以发送播放下一首歌的信息。

### `ServerSongs`

- 客户端建立连接的时候
- 回应 `ClientEnd` 的时候
- 服务端添加了一首歌的时候
- 服务端取消了一首歌的时候

服务端告诉客户端当前歌单。

### `ServerEndResult`

- 客户端结束一首歌，又出现状况的时候

服务端告诉客户端结束一首歌这个操作的结果。

### `ServerCancelResult`

- 服务端取消一首歌的时候。

服务端告诉客户端这次取消操作的结果。
